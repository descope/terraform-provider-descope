# descope_outbound_scim_configuration (Resource)

Manages one outbound SCIM configuration identified by its federated SAML or OIDC application ID.

The backend accepts a JSON configuration object so connector-template fields can evolve without requiring a provider release. Keep all non-secret values in `configuration`. Put bearer tokens, passwords, client secrets, and similar values in the write-only `secrets` JSON object; it is deep-merged into `configuration` for create and update requests and is never stored in Terraform state.

## Example Usage

```hcl
variable "scim_bearer_token" {
  type      = string
  sensitive = true
}

resource "descope_outbound_scim_configuration" "example" {
  project_id = "<project-id>"
  app_id     = "<federated-application-id>"
  enabled    = true

  configuration = jsonencode({
    host = "https://scim.example.com/v2"
    authentication = {
      method = "bearerToken"
    }
    ignoreUnverifiedEmails = false
    ignoreUnverifiedPhones = false
  })

  secrets = jsonencode({
    authentication = {
      bearerToken = var.scim_bearer_token
    }
  })
}
```

Import using `project_id/app_id`:

```shell
terraform import descope_outbound_scim_configuration.example <project-id>/<federated-application-id>
```

## Schema

### Required

- `app_id` (String) Federated SAML or OIDC application ID to provision from.
- `project_id` (String) Descope project ID that owns the outbound SCIM configuration.

### Optional

- `configuration` (String) JSON object containing non-secret connector configuration. Put token, password, and secret fields in `secrets`.
- `enabled` (Boolean) Whether outbound SCIM provisioning is enabled.
- `secrets` (String, Sensitive) Write-only JSON object deep-merged into `configuration` for create and update requests. Secret and token values are never stored in Terraform state.

### Read-Only

- `failures` (Number) Current consecutive failure count.
- `id` (String) Resource identifier, equal to the federated application ID.
- `last_export_time` (Number) Unix timestamp of the last export.
- `last_processing_time` (Number) Unix timestamp of the last processing run.
- `version` (Number) Configuration version used for optimistic concurrency.
